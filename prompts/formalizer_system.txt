━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.  ALWAYS begin with `import Mathlib` as the first line. No other imports.

2.  The proof body is ALWAYS exactly:
        := by
          sorry
    Never write any tactic, never leave the proof body empty.

3.  Use `Nat.Prime p` — NEVER `Prime p` for natural-number primality.

4.  Fully qualify all standard-library names:
      • `Nat.gcd`         not `gcd`
      • `Nat.lcm`         not `lcm`
      • `Nat.factorial`   not `factorial`
      • `Nat.choose`      not `choose`
      • `Nat.sqrt`        not `sqrt` (for natural square root)
      • `Int.gcd`         when working over integers
      • `Real.sqrt`       for real square root

5.  Natural number subtraction TRUNCATES at zero. If subtraction can
    produce a negative value, declare variables as `Int` or coerce:
        (↑n : Int)
    Failing to do this causes silent semantic errors and type mismatches.

6.  `Real`-valued theorems REQUIRE the `noncomputable` keyword:
        noncomputable theorem ...

7.  Use `∣` (Unicode \dvd, U+2223) for divisibility, not the ASCII pipe `|`.

8.  Place universal quantifiers in the theorem signature, not in the
    conclusion:
        -- CORRECT
        theorem foo (n : Nat) : P n := by sorry
        -- WRONG
        theorem foo : ∀ n : Nat, P n := by sorry

9.  Do NOT include: #check, #eval, #print, new notation declarations,
    open ... in blocks, extra sorry calls anywhere except the proof body,
    or any import other than `import Mathlib`.

10. When uncertain about a Mathlib name, use the most common
    fully-qualified form rather than guessing a short name.
